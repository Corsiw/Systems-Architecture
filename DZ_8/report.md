# Отчет по программе взаимодействия двух процессов Терехов Дмитрий Семинар 8

## Основные идеи реализации
* Используется **разделяемая память POSIX** (`shm_open`, `mmap`, `shm_unlink`);
* Есть **два независимых процесса (server и client)**;
* Взаимодействие идёт через **общую структуру `Shared` с кольцевым буфером** и **семафорами** (`sem_t`);
* Реализовано **корректное завершение** через общий флаг `g_stop`, установленный обработчиками `SIGINT`;
* При завершении одного процесса, **другой завершается автоматически** (через `shared->g_stop`);
* Сервер корректно **удаляет разделяемую память и семафоры**.

---

## Вариант корректного завершения.
При прерывании одного из процессов, другой через *Shared* *g_stop* получается информацию об остановке и корректно завершается.

**Server** владеет ресурсами, поэтому он управляет и очищает симофоры и другие данные. Также сервер корректно выводит все сгенерированные клиентом числа, даже при остановке одного из них.

---

## Сборка и запуск
1.  Сборка
    ```bash
    g++ -std=c++17 -pthread -O2 -o server src/server.cpp
    g++ -std=c++17 -pthread -O2 -o client src/client.cpp
    ```
2. Запуск
    ```bash
    ./src/server
    ./src/client
    ```
Запуск нужно производить в разных консолях. Ожидается что сначала запускается сервер, потом клиент.
